== Commands

.Nixos
[source, bash]
----
# Build system configuration
nixos-rebuild build --flake .

# Build and activate, with set new default
nixos-rebuild switch --flake .

# Activate temporary, non-permanent
nixos-rebuild test --flake .

# Create virtual-machine to test config
nixos-rebuild build-vm --flake .

# Dry build with a check of what will change
nixos-rebuild dry-activate --flake .

## Diffing
nix shell nixpkgs#nvd
sudo nixos-rebuild build --flake .

nvd diff /run/current-system ./result

nix store diff-closures /run/current-system ./result


# Garbage collecting
nix store gc # Removes dead paths, but keep all profiles

nix-collect-garbage # Removed dead paths and old profiles if -d specified.
----

== How to bootstrap

Depending on what OS is being used, either any Linux OS or NIXOS the bootstrapping is a bit different.

=== Other linux distro
Start by installing nix
[source,bash]
----
# Install nix
curl --proto '=https' --tlsv1.2 -sSf -L https://install.determinate.systems/nix | sh -s -- install

# Make sure flakes enabled  ~/.config/nix/nix.conf with the following line:
experimental-features = nix-command flakes
----

[IMPORTANT]
====
nixos-rebuild `--flake .` defaults to look for nixConfigurations named `hostname`  +
home-manager `--flake .` defaults to look for homeConfigurations named `$(USERNAME)@hostname`

Specify name explicitly when needed e.g. `--flake .#desktop` to use other than default, usefull if this is first setup and hostname is reconfigured later.
====


== First time use
[source, bash]
----
# Run this in repo root if NOT nix-command or flakes.
nix-shell

# If already has nix setup with flakes and nix-command, this run devShell in flake.nix
nix develop
----


.Other linux distro
[source, bash]
----
# Build system configuration
home-manager build --flake .engson@work

home-manager switch --flake .engson@work
----

== Update configs
[source, bash]
----
# Evaluate flake.nix and check if syntactically is correct
nix flake check

# Reconfigure flake after init 
nix flake update
# Git commit and push 
----

== Flakes
[source,bash]
----
nix flake archive - copy a flake and all its inputs to a store 
nix flake check - check whether the flake evaluates and run its tests 
nix flake clone - clone flake repository 
nix flake info - show flake metadata 
nix flake init - create a flake in the current directory from a template 
nix flake lock - create missing lock file entries 
nix flake metadata - show flake metadata 
nix flake new - create a flake in the specified directory from a template 
nix flake prefetch - download the source tree denoted by a flake reference into the Nix store 
nix flake show - show the outputs provided by a flake 
nix flake update - update flake lock file 
----
